## Summary of Changes

### Todo 009: HTTP Security Headers (next.config.ts)
Added an `async headers()` function to the `NextConfig` object that applies five security headers to all routes (`/(.*)`):
- `X-Frame-Options: SAMEORIGIN`
- `X-Content-Type-Options: nosniff`
- `Referrer-Policy: strict-origin-when-cross-origin`
- `Strict-Transport-Security: max-age=63072000; includeSubDomains`
- `Content-Security-Policy` — assembled from directives covering default, script, style, img, frame, connect, and font sources.

### Todo 020 Part 1: DATABASE_URL validation (src/db/index.ts)
Replaced the non-null assertion `process.env.DATABASE_URL!` with an explicit check that throws a descriptive error if the variable is missing. Also added `max: 1` to the Pool config for serverless compatibility (addresses Todo 014 as well).

### Todo 020 Part 2: Debounced previewTrade (src/components/TradePanel.tsx)
- Added `useRef` to the React import.
- Added `previewTimerRef = useRef<NodeJS.Timeout | null>(null)` alongside the other state declarations.
- Changed `handleAmountChange` from `async function` to a plain `function` (the async work is now inside the timer callback).
- Replaced the direct `await previewTrade(...)` call with a 400 ms debounce: any pending timer is cleared before scheduling a new one, so rapid keystrokes or preset-button clicks no longer fire a server action on every change.
- `handleOutcomeChange` retains its immediate `previewTrade` call (deliberate — outcome changes are explicit user actions, not continuous input).
