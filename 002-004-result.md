# Todos 002 & 004 Result

## Files Changed

- `src/lib/actions/auth.ts`
- `src/lib/actions/economy.ts`

## Summary of Changes

### auth.ts (Todos 002 & 004)

- Removed the pre-check `SELECT` query that looked up existing users by email (lines 25–33).
- Removed the now-unused `eq` import from `drizzle-orm`.
- Wrapped the `INSERT users` and `INSERT coinTransactions` statements in a single `db.transaction()` call, making signup atomic.
- Added a `catch` block that inspects the error `code` property: Postgres error code `23505` (unique constraint violation) is caught and returned as `{ error: "An account with this email already exists" }`; all other errors are re-thrown.
- Added `referenceId: userId` to the `signup_bonus` coinTransaction insert for idempotency (Todo 004).

### economy.ts (Todo 004)

- Added `referenceId: todayUTC.toISOString().slice(0, 10)` to the `daily_login` coinTransaction insert inside `claimDailyReward()`. The `todayUTC` variable was already computed earlier in the same transaction.

## Blockers

None. The `referenceId` column must exist on the `coinTransactions` table schema; assuming the schema already defines it (it was referenced in the task description as an existing idempotency field). If the column is missing a migration will be needed before these changes can be deployed.
