# Todo 014 & 015 — Result

## Files Changed

- `src/lib/auth.ts`
- `src/app/portfolio/page.tsx`

- `src/app/page.tsx`

## Summary of Changes

### Part A — auth.ts: Google OAuth signup bonus (Todo 015)

Added `coinTransactions` to the schema import. Implemented the previously empty `events.createUser` handler to insert a `signup_bonus` coin transaction (1000.00 coins) for new OAuth users. The insert is wrapped in try/catch so a unique constraint violation (already awarded) is silently ignored, making the operation idempotent.

### Part B — portfolio/page.tsx: UTC consistency for daily reward check (Todo 015)

Replaced the local-time `setHours(0,0,0,0)` comparison with UTC normalization using `Date.UTC(...)`. Both the "today" boundary and the stored `lastLoginReward` date are now normalized to midnight UTC before comparing, matching the behaviour of `claimDailyReward` in economy.ts.

### Part C — leaderboard/page.tsx: inArray filter on positions query (Todo 014)

Changed the parallel `Promise.all` into a sequential two-step query. The first query fetches up to 100 top users. Their IDs are extracted into `topUserIds`. The second query (positions) now includes `.where(and(ne(positions.shares, "0"), inArray(positions.userId, topUserIds)))`, limiting the positions scan to only the rows that are needed. Added `and` and `inArray` to the drizzle-orm import. The positions query is skipped entirely (returns `[]`) when there are no top users.

### Part D — page.tsx: limit active markets query (Todo 014)

Added `.limit(50)` after `.orderBy(desc(markets.createdAt))` on the active markets query to prevent unbounded full-table scans on the homepage.
